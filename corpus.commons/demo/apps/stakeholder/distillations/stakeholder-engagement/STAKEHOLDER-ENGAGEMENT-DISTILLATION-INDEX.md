# Stakeholder-Engagement Distillation Index

> **Purpose:** Situation-to-resource router for stakeholder-engagement work. "In phase X of an engagement, which references should I reach for?"

This is the *when-to-use* layer for the stakeholder-engagement task. It partitions the task into phases and lists distillations appropriate to each phase. For the *what-is* layer (descriptions of each reference), read [`reference-index.json`](../../reference-index.json) at the corpus root.

**Workflow:** Distillation index (when) → distillation (how) → deep reference (citations, if needed).

---

## Format

Two table shapes appear in this index.

**Phase-by-phase tables** (the main routing layer) use four columns. The "Need" column describes the situation; "Reference" names the source; "Distillation" links to the projected distillation; "Reach for this when" gives a one-line routing cue.

```markdown
| Need | Reference | Distillation | Reach for this when |
|---|---|---|---|
| ... | ... | ... | ... |
```

**Quick-start tables** (the situation-first lookup) use three columns: a situation description, the primary references that apply, and the distillation file paths.

```markdown
| Engagement type | Primary references | Distillations |
|---|---|---|
| ... | ... | ... |
```

### Doctrinal frame for multi-stakeholder engagement

- `mcdp1-warfighting-stakeholder-engagement.md`, U.S. Marine Corps (*MCDP 1: Warfighting*, 1997, supersedes FMFM-1 1989). **Public domain (US Government work product); scope=open.** MCDP-1 is doctrinally about combat — adversarial engagement between two hostile wills — so most threads do not transfer cleanly to civilian stakeholder work. Three threads do: (1) *intent must be understood two levels up* (Ch 4) — the operative test for stakeholder engagement at any scale is *do the stakeholders know the intent two levels above the work they are doing?* Intent travels; process does not; (2) *main effort / Schwerpunkt* (Ch 4) — when stakeholder interests conflict, the discipline is *name the main effort; every other interest is supporting effort that yields when in conflict*; some stakeholders being explicitly subordinated is operationally necessary; (3) *surfaces and gaps + reconnaissance pull* (Ch 2, Ch 4 Notes 11) — engage where receptivity is high; do not push against entrenched positions; let the encountered openings shape the engagement's main effort. *Doctrine is a way of thinking, not a checklist* (Ch 3) applies to stakeholder-engagement methods themselves. Caution: the *Zweikampf* (war as violent struggle between two hostile irreconcilable wills) framing rarely fits civilian stakeholder contexts; most stakeholders are negotiable parties whose interests partly align. Reach for MCDP-1 in high-conflict situations (regulatory enforcement, contract disputes, public-policy battles, hostile takeovers) where adversarial framing is actually operative; otherwise use the framework selectively.

## Anti-patterns this index helps avoid

- Treating standing as a function of seniority rather than of who is affected by the decision.
- Convening one large meeting to do constraint-discovery, synthesis, and ratification in sequence.
- Letting late-stage stakeholders reopen a closed decision without asking whether they had standing originally.
- Running a synthesis workshop when the team already knows the answer and only needs ratification.
