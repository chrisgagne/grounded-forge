# Decision-Making Distillation Index

> **Purpose:** Situation-to-resource router for decision-making work. "In phase X of a decision, which references should I reach for?"

This is the *when-to-use* layer for the decision-making task. It partitions the task into phases and lists distillations appropriate to each phase. For the *what-is* layer (descriptions of each reference), read [`reference-index.json`](../../reference-index.json) at the corpus root.

**Workflow:** Distillation index (when) → distillation (how) → deep reference (citations, if needed).

---

## Format

Two table shapes appear in this index.

**Phase-by-phase tables** (the main routing layer) use four columns. The "Need" column describes the situation; "Reference" names the source; "Distillation" links to the projected distillation; "Reach for this when" gives a one-line routing cue.

```markdown
| Need | Reference | Distillation | Reach for this when |
|---|---|---|---|
| ... | ... | ... | ... |
```

**Quick-start tables** (the situation-first lookup) use three columns: a situation description, the primary references that apply, and the distillation file paths.

```markdown
| Decision type | Primary references | Distillations |
|---|---|---|
| ... | ... | ... |
```

Concepts within a distillation may be referenced with line ranges (`{file}.md L{start}-L{end}`) for fine-grained pointers. The build validates the filename inside the backticks; line ranges are a human convention for locating the relevant passage and are not enforced by the build.

## Anti-patterns this index helps avoid

- Convening stakeholders before bounding the decision.
- Debating visible constraints while the binding one sits unnamed.
- Treating decisions as artefacts rather than hypotheses to be revisited.
- Routing by author name (read this *because* author X wrote it) rather than by decision phase (read this *because* you are in phase Y).
