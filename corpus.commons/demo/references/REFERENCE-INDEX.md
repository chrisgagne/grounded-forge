# Reference Index

> **Purpose:** Corpus catalogue. Find a source by author, topic, or concept. Answer the question "is there a reference in this library on X?"

This index is the *what-is* layer. It describes what each source is about. For *when-to-use* routing (which reference to reach for in a particular phase of a task), use the per-task distillation indexes:

- `distillations/{task}/{TASK-UPPERCASE}-DISTILLATION-INDEX.md` — one per task axis the compiled app ships. The demo corpus has five: decision-making, stakeholder-engagement, software-business, aar, retro. Each shipped app carries the index for its profile's axis only.

**Workflow:** REFERENCE-INDEX (what) → distillation index (when) → distillation (how) → deep reference (citations).

This index has two sections. The first lists every reference in the corpus by file. The second indexes concepts A-Z, with each concept entry listing sub-aspects pointing to file plus optional line range.
